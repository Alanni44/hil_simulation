# HIL 全流程契约审查设计

## 目标

在 Ubuntu 18.04、MATLAB R2018b（含 Embedded Coder）、Python 3.6.9 和 GCC 7.x 上，使测试模型和符合严格根端口契约的上传模型能够完整经过分析、ERT 出码、C 运行时编译、服务启动及 UDP 集成测试。

## 范围

本次审查覆盖 MATLAB 构建脚本、生成模型、C 运行时、Python 服务、Shell 测试脚本和静态回归测试。每个修复必须对应日志证据或可复现的跨组件契约缺陷；不重写用户上传模型的拓扑。

## 构建契约

1. ERT 生成目录、缓存目录和 MATLAB 当前目录必须相互分离，并由构建脚本显式设置。
2. 构建脚本以 `RTW.getBuildDir(...).BuildDirectory` 作为生成代码目录的唯一来源。
3. GCC 只能编译模型算法源码和项目运行时源码；必须排除 ERT 自带入口 `ert_main.c`，以保证可执行文件只有 `main_rt.c` 的一个 `main`。
4. 生成桥接头通过 `-I<代码目录>` 和未加引号的预处理器 token 引用，禁止把绝对路径经多层字符串化传入 `#include`。
5. 任何 MATLAB、ERT 或 GCC 失败必须写入结果 JSON，并由集成脚本打印其错误消息后退出。

## 接口契约

1. 上传模型必须包含可映射到 `pos_x`、`pos_y`、`pos_z` 的标量数值根输出；缺失即失败。
2. 根输入和输出必须为标量数值或布尔类型；数组、Bus、枚举和定点接口拒绝构建。
3. JSON 解码后的端口列表可为结构体数组或单元数组。
4. ERT 头文件中的所有 `ExtU_*` 和 `ExtY_*` 字段均必须被解析，即使每个字段后有 `/* ... */` 或 `// ...` 注释。
5. 字段映射、桥接头和安全读写宏必须只引用生成头文件中实际存在的字段。

## 运行时契约

1. C 运行时使用项目自己的 `main_rt.c` 作为唯一入口；不调用或链接 ERT 的示例 main。
2. 命令线程与模型步进线程通过既有无锁序列快照交换输入；主线程只在 `model_step()` 前应用稳定快照。
3. 生成模型的外部输入和外部输出访问必须通过桥接 typedef 和安全宏，不依赖结构体偏移猜测。
4. 缺少可选遥测输出时使用既定默认值；必需位置输出不允许默认替代。

## Python 与脚本契约

1. Python 源码必须可由 Python 3.6 语法解析，依赖固定为 `PyYAML==6.0.1`。
2. 集成脚本传递实际 `.slx` 路径，清除上一次结果 JSON，并在任一阶段失败时显示可行动的原因。
3. 集成测试的二进制字段索引与 `flight_state.h` 一致，服务 PID 检查使用存活性而非脆弱的文本匹配。

## 验证策略

Windows 上执行静态回归、Python 3.6 语法解析、补丁检查和 C 源码入口/生成文件白名单审查。Ubuntu 目标机执行一次 `bash scripts/integration_test.sh` 作为唯一的 MATLAB R2018b、GCC 7.x 与运行时集成实证。

## 非目标

不改变任意上传 `.slx` 的控制逻辑；不把测试模型中未使用的可选命令或物理参数伪装成已实现功能；不为不符合标量根端口契约的模型添加隐式转换。
